Quaternion.Euler

In Unity, Quaternion.Euler is a static function that creates a Quaternion (a mathematical representation of 3D rotation) from Euler angles (the three angles that represent a 3D rotation in terms of pitch, yaw, and roll). The function takes in three parameters, which are the Euler angles for the rotation around the x-axis, y-axis, and z-axis, respectively. It returns a Quaternion that represents the rotation.

Here's an example of how to use the Quaternion.Euler function to create a Quaternion for a 90-degree rotation around the y-axis: 

Quaternion rotation = Quaternion.Euler(0, 90, 0);

In Unity, you can get a reference to another script component on the same GameObject or on another GameObject using the GetComponent function.

Here's an example of how you could use GetComponent to get a reference to another script component: 

public class MyScript : MonoBehaviour
{
    private OtherScript otherScript;

    private void Start()
    {
        // Get a reference to the OtherScript component on the same GameObject
        otherScript = GetComponent();
        
        if (otherScript == null)
        {
            Debug.LogError("OtherScript component not found on the same GameObject as MyScript.");
        }
    }
}

In this example, the Start function uses GetComponent to get a reference to the OtherScript component on the same GameObject as MyScript. If the OtherScript component is not found, the Debug.LogError function is called to print an error message to the console.

Get The Script From Other GameObjects

You can also use GetComponent to get a reference to a script component on another GameObject by passing in a reference to the GameObject that has the component: 

public class MyScript : MonoBehaviour
{
    public GameObject otherGameObject;
    private OtherScript otherScript;

    private void Start()
    {
        // Get a reference to the OtherScript component on the otherGameObject GameObject
        otherScript = otherGameObject.GetComponent();

        if (otherScript == null)
        {
            Debug.LogError("OtherScript component not found on the otherGameObject GameObject.");
        }
    }
}

It's generally best to use GetComponent to get a reference to a script component, rather than directly accessing the script or using the GameObject reference to access the script.
